## 3. 確認できる要素（ご要望への対応）

### 🔵 React の強みがわかるポイント
- **即時性の高いUI (モダンなデザイン)**: 在庫一覧画面の検索バーに文字を入力すると、サーバー通信なしで**即座にテーブルがフィルタリング**されます。
- **動的フォーム (条件付きレンダリング)**: 在庫受入・払出モーダルにて、選択した製品マスタのカテゴリ（通常品、生鮮食品、危険物）に応じて、**自動的に必要な入力フォーム（消費期限や承認チェックボックス）がスッと表示**されるアニメーションが体験できます。

### 🔵 ドメイン駆動設計 (DDD) の強みがわかるポイント
- `src/domain/entities/Product.ts` をご覧ください。
- 製品のカテゴリー別に**ビジネス検証ルールをエンティティ自身（クラス）内にカプセル化**（ポリモーフィズム）しています。
- これにより、「危険物の出庫時にチェックを忘れる」といった業務エラーを、**UIやUseCaseの if/else ではなく堅牢なDomain層**が自動で防ぎます。

### 🔵 クリーンアーキテクチャ
- 本来別々にリポジトリを作るフロントエンドとAPIサーバーを Next.js App Router で一つにまとめつつ、`src/` 配下を厳密に `domain`, `usecases`, `adapters`, `app` と分割し、依存性注入(DI)で結合しています。データベースがインメモリか実DBかにかかわらず、ロジックは影響を受けません。
