# Docker化の完了

アプリケーションをコンテナ化して実行するための全ての準備が整いました。効率的で軽量なビルド設定を導入しています。

## 実施内容

### 1. Next.js ビルド設定の最適化
- `next.config.ts` に `output: 'standalone'` を追加しました。これにより、実行時に不要な依存関係を排除し、イメージサイズを劇的に削減できます。

### 2. コンテナ化設定ファイルの作成
- **Dockerfile**:
  - `deps`, `builder`, `runner` の3ステージによるマルチステージビルドを採用。
  - セキュリティとイメージサイズの観点から Node.js 20-alphine をベースイメージに使用。
  - ルート権限以外で実行するよう `nextjs` ユーザーを設定。
- **.dockerignore**:
  - ビルドに不要な `node_modules`, `.next`, ドキュメント類などを除外し、ビルド時間の短縮とイメージの軽量化を図りました。
- **docker-compose.yml**:
  - `docker compose up --build` コマンド一発でビルドから起動までを自動化できるようにしました。

## 実行方法

以下のコマンドを実行することで、アプリケーションがコンテナ上で起動します。

```bash
docker compose up --build
```

起動後、ブラウザで `http://localhost:3000` にアクセスして動作を確認してください。
